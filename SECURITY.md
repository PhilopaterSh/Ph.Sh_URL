# Security Policy

## Supported Versions

We are committed to ensuring the security of Ph.Sh_URL. Security updates are provided for the most recent version.

| Version | Supported          |
| ------- | ------------------ |
| 1.1.x   | :white_check_mark: |
| < 1.1   | :x:                |

## Reporting a Vulnerability

We take all security vulnerabilities seriously. Thank you for taking the time to responsibly disclose any issues you find.

To report a security vulnerability, please use the **"Report a vulnerability"** button on the "Security" tab of our GitHub repository.

**Please do not report security vulnerabilities through public GitHub issues.**

When submitting your report, please include the following details:

- A clear and concise description of the vulnerability.
- The steps required to reproduce the vulnerability.
- A proof-of-concept, if possible.
- Any relevant logs, screenshots, or other information.

You can expect to receive an acknowledgment of your report within **48 hours**. We will keep you updated on our progress as we investigate and work on a fix. We appreciate your contribution to keeping our project secure.
